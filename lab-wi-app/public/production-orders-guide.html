What a Production Order is

A Production Order is one run of an approved Work Instruction — making a specific lot of a reagent. The order turns the WI's steps into a guided, step-by-step checklist the operator works through on the bench: it captures real weights and volumes, enforces tolerances, times mixes, records timestamps, flags deviations, and finishes with QC and a Certificate of Analysis. Everything you do is logged against the lot for a complete production record.

Your role today

▶
Operator
Runs the order on the floor — works each step, weighs/dispenses, records QC, flags deviations.
＋
Approver / Admin
Creates production orders from approved WIs (and, later, from D365 planned orders); can assign, cancel, or delete.
Only an approved Work Instruction can be produced. If you can't start a run, check that its WI has cleared the approval workflow first.

Walkthrough

1

Create the production order

ApproverAdmin
  1. From an approved Work Instruction: open it and click Start Production — the fastest route, since it pre-links the WI.
  2. Or from the Production Orders list: use New Production Order and pick the approved WI.
  3. Enter the lot number and the batch size + unit for this run. (Orders pulled from D365 planned orders arrive pre-filled.)
  4. Create it — the order starts in Pending, ready to run.
Each attendee makes their own order for their WI, so everyone has a run to drive.
2

Stage the bench & start the run

Operator
  1. Review the Materials summary — the bill of materials rolled up from every step, shown up top so you can gather everything before you begin.
  2. Confirm who's running it in Assigned to (you can hand it off later — see Task 6).
  3. Click Start Production. The status moves to In Progress, a progress bar appears, and the steps unlock.
D365-linked orders post a ProductionOrderStart message to D365 when you start — you'll see a small status panel confirm it sent (manual orders skip this).
3

Work the steps — the Happy Path

Operator

Steps run one at a time. The step navigator on the left shows the whole run with a status dot per step — click any step to jump to it. Open a step, do what it asks, then Complete Step (or Skip). Guided steps:

⚖
Weigh
Scan/select your bench scale, take the live reading — it must be in tolerance or it won't let you continue. Enter the lot number if it's lot-controlled.
💧
Dispense
Same idea for a volume — measure to target ± tolerance; out-of-range is blocked.
⏱
Mix
Run the built-in timer to the target duration before completing.
🕐
Record Time
Tap to stamp the current time (e.g. water-bath start / end).
👁
Observe / Notes
Record a verification result or free-text note.
📎
Attachment
Upload supporting docs; if marked required, you must attach one to finish.
Went too fast? Reopen a completed step to correct it — the run drops back to In Progress if it had finished.
4

Flag a deviation

Operator
  1. On a Possible Deviation step (or when something looks off-spec), record the impacted quantity.
  2. Hit Notify Supervisor — the platform fires a Microsoft Teams message to the supervisor ("technician requests assistance" for this order).
  3. Carry on, or wait for guidance — the deviation is captured on the production record either way.
Demo tip: this is the "play with it" moment — trigger a deviation to see the supervisor notification fire end-to-end.
5

QC checks & finishing the run

OperatorApprover
  1. When the steps are done, the Quality Control panel lists the product's release tests. Enter each result — the system checks it against the spec limits (pass / fail / in-range) as you go.
  2. All steps + all QC results in → the order flips to Completed. If QC results are still outstanding, it sits at Awaiting QC until they're entered.
  3. Open the Certificate of Analysis to see the finished, signed-off record for the lot.
6

Hand off & send back (optional)

Operator
  1. Pulled away mid-run? In Assigned to, pick another person, add an optional note (e.g. "picked up through step 6, please finish QC"), and Hand off. They get notified and the run keeps its progress.
  2. When they're done with their part, they click ↩ Send back to return the run to whoever handed it over. It can bounce back and forth.
Every step still records who actually completed it, so the production record stays accurate across hand-offs.

Order statuses at a glance

PendingCreated but not started. Materials can be staged; nothing captured yet.
In ProgressStarted — steps are being worked. The progress bar tracks steps completed.
Awaiting QCAll production steps finished, but one or more QC release results are still outstanding.
CompletedSteps and QC all done. Certificate of Analysis available.
CancelledStopped by an admin or the creator. No longer runnable.

If you get stuck

The "Complete Step" button is greyed out — why?
The step's requirement isn't met yet. Common causes: a Weigh with no scale connected, or a reading out of tolerance; a lot-controlled material missing its lot number; a required attachment not uploaded; or a pH reading not in range. Fix the flagged item and the button turns on.
My weight is out of tolerance and I can't continue.
That's by design — an out-of-tolerance measurement blocks completion so an off-spec batch can't move forward. Re-weigh until you're inside the allowed range, or raise a deviation if the material itself is the problem.
I can't start a run.
Two things to check: the underlying Work Instruction must be Approved, and the order must be in Pending status. A run that's already In Progress won't show the Start button.
Who can create a production order?
Approvers and Admins (and authors) create orders — from an approved WI's Start Production button or the New Production Order screen. Operators run the orders once they exist.
I completed a step by mistake.
Open that step and click Reopen. It returns to editable, and if the whole order had been marked complete it drops back to In Progress so you can redo it.
How do I get help from a supervisor mid-run?
Use a Possible Deviation step (or flag one), record the impacted quantity, and press Notify Supervisor — it sends a Microsoft Teams message so they can weigh in without you leaving the bench.
