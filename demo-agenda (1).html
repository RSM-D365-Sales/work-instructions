Session Schedule — Day 1 (Wednesday, July 22)

10:00

Welcome, Overview, Login, Roles & Security

70 min · to 11:10 RSM Lead: ________________
RSM Overview
  • Introductions, goals for the day, and how the session will run
  • The big picture: Dashboard tour and the reagent-production lifecycle — Work Instruction → Approval → Production Order → Execution → QC → Scheduling
  • How the platform maps to today's Uniflow process (and what's changing)
  • Login & the role model — Admin, Author, Approver, Operator, Lab — and what each can see and do
  • User management, labs, and equipment setup (Admin view); the Notifications inbox; default-lab selection
Hands-On — everyone signs in, in role
AdminAuthorApproverOperatorLab
  • Each user logs in with their assigned role and explores which menus appear
  • Test the boundaries: an Operator can't edit Work Instructions; an Author can't approve their own; only Admin sees Users / Equipment / Labs
  • Admin: quick look at creating a user and assigning a role
Q&A / Wrap-up
  • Set expectations — what does a successful day look like? Does this role & security model fit your segregation-of-duties needs?
11:20

Work Instructions Core

70 min · to 12:30 RSM Lead: ________________ 📘 Help Guide
RSM Overview
  • Anatomy of a Work Instruction — product / reagent-item link, target molarity, scheduled time
  • The Step Library and step types: gather reagents, weigh, dispense, mix, stir/vortex/invert, pH adjust, heat/cool, transfer, bring-to-volume, cap, package, attachments, QC, and more
  • The editor experience — step navigator, insert-a-step-between, collapse/expand, drag-to-reorder
  • Versioning, the version diff view, and the approval workflow
Hands-On — build & approve
AuthorApprover
  • Each user picks their own existing Work Instruction (e.g. Acidified Methanol, NSE Muscles Phosphate Buffer) — or creates a new one
  • Walk through the steps together; add or edit a step to see the authoring tools
  • Send for approval
  • Each person approves someone else's — cross-review so no one approves their own work
Q&A — likes & gaps
  • What do you like? What are you missing? This is a rich discussion area — we'll capture everything for the wrap-up review.
12:30 🍽 Lunch — 45 min · to 1:15
1:15

Production Orders — Execution Core

60 min · to 2:15 RSM Lead: ________________ 📘 Help Guide
RSM Overview
  • Creating a production order from an approved Work Instruction (and, later, from D365 planned orders)
  • The execution experience — step-by-step guidance, scale scan & weigh tolerance gating, volumetric dispense, attachments
  • Possible-deviation flagging → notify supervisor (Microsoft Teams)
  • End-of-run QC tests, order statuses (awaiting QC, complete), and the Certificate of Analysis
Hands-On — run a production order
OperatorApprover
  • Each user manually creates their own production order for their Work Instruction
  • Happy Path — execute the run end-to-end, weighing / dispensing within tolerance
  • Deviation — play with it — flag a possible deviation and trigger the supervisor notification
  • QC Checks — complete the quality checks and review the result / certificate
Q&A — Recap
  • Recap the run: what felt natural, what would you change on the floor?
